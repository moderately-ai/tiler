Tom decided on 2026-07-23 that typed explain infrastructure stays a compiler-owned module rather than becoming a `tiler-explain` crate. File the next sequential ADR recording it, and represent the explain authority in docs/compiler/optimizer.md, which owns Explainability.

The decision and its supporting evidence, all verified against the merged implementation:

- `tiler_compiler::explain` remains a module of `tiler-compiler`. It is private today and may be promoted to `pub` when the public compiler API requires it; promotion is not itself a crate decision.
- Extraction today would invert the dependency graph. `explain.rs` imports `crate::fusion::FusionNumericalProof` and `crate::request::{VerifiedTargetRequest, LoweringProviderIdentity}`, so a `tiler-explain` crate would depend on `tiler-compiler` while `tiler-compiler` depends on it to emit records. Extraction would first require relocating those subject types.
- There is no second consumer. `tiler-artifact` depends only on `tiler-ir`, docs/artifact-abi.md never contemplates embedding explain traces, and only 2 of `explain.rs`'s 3,047 lines reference `tiler_ir`.
- ADR 0070 had just consolidated shared compiler IR into `tiler-ir` and dropped the compiler-to-artifact edge; minting a new crate immediately afterward would cut against that consolidation.

Record the reconsideration trigger explicitly, because it is the durable part: if a second crate must ever READ canonical traces, the record/subject/disposition vocabulary moves into `tiler-ir` following the `AbiExpr` co-location precedent of ADRs 0068 and 0070, with emission staying compiler-owned. A new crate is not the expansion path. Cite the merged typed-explain implementation as evidence and cross-reference ADRs 0068 and 0070.

This ticket records an accepted decision; it does not reopen it. Do not move code.

## Outcome

[ADR 0073](../docs/decisions/0073-own-typed-explain-in-tiler-compiler.md), "Own
typed explain infrastructure in tiler-compiler", records the decision. It
refines ADR 0070, cross-references ADR 0068's `AbiExpr` co-location precedent,
and cites the crate-layout and general-compilation-boundary research records as
evidence. `docs/compiler/optimizer.md` gains an "Explain authority" subsection
under Explainability naming the compiler-owned vocabulary, compiler-owned
emission, the visibility/packaging distinction, and the reconsideration trigger.

Every evidence claim was re-verified against the merged implementation rather
than taken from this ticket:

- `mod explain;` is declared privately at `crates/tiler-compiler/src/lib.rs:6`,
  and `explain.rs` contains no bare `pub` item; the module is `pub(crate)`
  throughout.
- `crates/tiler-compiler/src/explain.rs:11-12` imports
  `crate::fusion::FusionNumericalProof` and
  `crate::request::{LoweringProviderIdentity, VerifiedTargetRequest}`, defined
  at `fusion.rs:66`, `request.rs:89`, and `request.rs:214`. `pipeline.rs` emits
  through `crate::explain`, so extraction would need both
  `tiler-explain -> tiler-compiler` and `tiler-compiler -> tiler-explain`.
- `crates/tiler-artifact/Cargo.toml` declares `tiler-ir` as its only
  dependency; no explain identifier appears anywhere in `tiler-artifact`,
  `tiler-ir`, or `tiler-metal`; and `docs/artifact-abi.md` mentions "explain"
  only inside "versioned, explainable" at line 336, never as embedded trace
  content.
- `explain.rs` is 3,047 lines and mentions `tiler_ir` on exactly two of them,
  lines 2131 and 2135, both inside the `#[cfg(test)]` module that begins at
  line 2126.

The generated thematic and chronological blocks in `docs/decisions/README.md`
were regenerated by `scripts/docs.py render`, which is why this branch touches a
`contracts/navigation` file. `contracts/navigation` is declared as a shared
scope, matching how other catalog-regenerating tickets carry it: this branch
authors no navigation content and only preserves the regenerated blocks the gate
requires.

The ADR's `applies_to` names `tiler.contract.optimizer` alone. The crate-graph
consequence is already governed in `docs/architecture.md` by ADR 0070, whose
decision text places explanations in `tiler-compiler`; the architecture
component table does not enumerate explainability among `tiler-compiler`'s
responsibilities, which a `contracts/foundation` pass could tighten. Claiming
`applies_to` over a contract this branch cannot edit would assert authority the
document does not represent.
